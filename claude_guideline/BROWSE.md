# Browse Page — Feature Reference

This document covers the Browse page (`view-browse`): entry display, quick filter pills,
stats bar, sorting, and search.

---

## 1. Data model

Browse entries come from `/api/browse` and are structured as:

```js
{
  day:         "241015",       // shooting day string (shown as a pill)
  scenes:      ["12", "12A"],  // one or more scene strings
  code:        "EXT_NIGHT",    // underscore-separated codes → split into pills
  description: "Rooftop...",   // plain text label
  slate_count: 7,              // total number of slates under this entry
  has_data:    true,           // whether any raw data exists
  subdirs:     [...]           // nested sub-entries (same shape)
}
```

On the client, `data.by_days`, `data.by_scenes`, `data.by_codes` are the three pre-grouped
views used by grouping modes (`days`, `scenes`, `codes`). Ungrouped mode (`none`) uses
`data.by_days` flattened.

## 2. Stats bar

Three counters shown at the top of the Browse view:

```html
<div class="stat"><div class="stat-value" id="stat-total">0</div>  <div class="stat-label">Entries</div></div>
<div class="stat"><div class="stat-value" id="stat-groups">0</div> <div class="stat-label">Groups</div></div>
<div class="stat"><div class="stat-value" id="stat-slates">0</div> <div class="stat-label">Slates</div></div>
```

`updateStats(total, groups, slates)` writes to all three:
```js
function updateStats(total, groups, slates) {
    document.getElementById('stat-total').textContent  = total;
    document.getElementById('stat-groups').textContent = groups;
    document.getElementById('stat-slates').textContent = slates || 0;
}
```

- **Entries**: number of visible entries after filtering.
- **Groups**: number of non-empty groups (0 in ungrouped mode).
- **Slates**: sum of `entry.slate_count` across all matched entries.

## 3. Quick filter pills

### State
```js
let activeFilterType  = null;   // 'day' | 'scene' | 'code' | null
let activeFilterValue = null;   // e.g. '241015' | '12A' | 'EXT' | null
```

### Core functions

| Function | Action |
|---|---|
| `setQuickFilter(type, val)` | Sets `activeFilterType`/`activeFilterValue`; if already active → clears (toggle); calls `render()` |
| `clearQuickFilter()` | Clears both state vars; calls `render()` |
| `entryPassesFilter(e)` | Returns `true` if no filter, or if the entry matches the active filter |

```js
function entryPassesFilter(e) {
    if (!activeFilterType) return true;
    if (activeFilterType === 'day')   return e.day === activeFilterValue;
    if (activeFilterType === 'scene') return (e.scenes || []).includes(activeFilterValue);
    if (activeFilterType === 'code')  return (e.code || '').split('_').includes(activeFilterValue);
    return true;
}
```

Quick filter is **AND**'d with text search — both must pass for an entry to be shown.

### Filter chip (info bar)

When a filter is active, a dismissible chip appears in `#search-info`:

```
  day: 241015 ✕         (green)
  scene: 12A ✕          (cyan)
  code: EXT ✕           (amber)
```

Generated by `_filterChip()` inside `render()`, inserted via `infoEl.innerHTML`:
```js
function _filterChip() {
    if (!activeFilterType) return '';
    return `<span class="browse-filter-chip chip-${activeFilterType}"
                  onclick="clearQuickFilter()">
                ${escHtml(activeFilterType)}: <strong>${escHtml(activeFilterValue)}</strong> ✕
            </span>`;
}
```

**Note:** `infoEl.innerHTML` (not `.textContent`) is used so the chip HTML is parsed.
The search query text is `escHtml()`-escaped before insertion.

## 4. Entry layout (`renderEntry`)

### Single-line title

All elements sit on one flex row (`.entry-title-line`) with `flex-wrap: wrap`.
Order: day pill → scene pill(s) → code pill(s) → description (flex:1).

### Pill sizing

```css
.title-day, .title-scene, .title-code {
    padding: 2px 7px;      /* was 3px 9px */
    font-size: 0.75em;     /* was 0.8em */
    cursor: pointer;
    transition: filter 0.12s, border-color 0.12s, background 0.12s;
}
```

### Clickable pills with filter state

Every day/scene/code pill carries `data-filter-type` and `data-filter-val` attributes,
and a `.filter-active` class when its value matches the current filter.
The onclick uses `this.dataset.*` to avoid single-quote quoting issues:

```js
// Day pill
const dayActive = (activeFilterType === 'day' && activeFilterValue === entry.day) ? ' filter-active' : '';
const dayHtml = `<span class="title-day${dayActive}"
    data-filter-type="day"
    data-filter-val="${escHtml(entry.day)}"
    onclick="setQuickFilter(this.dataset.filterType, this.dataset.filterVal)"
    title="Filter by day">${highlight(entry.day, q)}</span>`;

// Scene pills (one per scene)
const sceneHtml = (entry.scenes || []).map(s => {
    const active = (activeFilterType === 'scene' && activeFilterValue === s) ? ' filter-active' : '';
    return `<span class="title-scene${active}"
        data-filter-type="scene"
        data-filter-val="${escHtml(s)}"
        onclick="setQuickFilter(this.dataset.filterType, this.dataset.filterVal)"
        title="Filter by scene">${highlight(s, q)}</span>`;
}).join('');

// Code pills (split on '_')
const codeHtml = (entry.code || '').split('_').map(c => {
    const active = (activeFilterType === 'code' && activeFilterValue === c) ? ' filter-active' : '';
    return `<span class="title-code${active}"
        data-filter-type="code"
        data-filter-val="${escHtml(c)}"
        onclick="setQuickFilter(this.dataset.filterType, this.dataset.filterVal)"
        title="Filter by code">${highlight(c, q)}</span>`;
}).join('');
```

### Description

Plain bold text, no pill background:
```js
const descHtml = `<span class="entry-description">${highlight(entry.description || '', q)}</span>`;
```
`.entry-description` has `font-weight: 600; flex: 1` so it fills the remaining row width.

## 5. `render()` internals

Three helpers defined inside `render()` on each call:

```js
function _applyFilters(arr) {
    return arr.filter(e => (!q || entryMatches(e, q)) && entryPassesFilter(e));
}
function _totalSlates(arr) {
    return arr.reduce((s, e) => s + (e.slate_count || 0), 0);
}
function _updateInfo(shown, total) {
    const parts = [];
    if (q) parts.push(`Showing ${shown} of ${total} matching …`);
    const chip = _filterChip();
    if (chip) parts.push(chip);
    infoEl.innerHTML = parts.join(' &ensp;·&ensp; ');
    infoEl.style.display = parts.length ? 'block' : 'none';
}
```

Ungrouped mode (`none`):
```js
const all     = Object.values(data.by_days || {}).flat().sort(…);
const matched = _applyFilters(all);
updateStats(matched.length, 0, _totalSlates(matched));
_updateInfo(matched.length, all.length);
```

Grouped mode:
```js
for (const [key, entries] of Object.entries(modeData)) {
    const matched = _applyFilters(sorted);
    if (!matched.length) continue;
    groupsShown++;
    totalShown  += matched.length;
    totalSlates += _totalSlates(matched);
    …
}
updateStats(totalShown, groupsShown, totalSlates);
_updateInfo(totalShown, grandTotal);
```

## 7. Important implementation notes

### `data-filter-type` / `data-filter-val` pattern
Pill onclick handlers use `this.dataset.filterType` / `this.dataset.filterVal` to retrieve
the filter values, rather than embedding them directly in the onclick string.
This avoids HTML attribute single-quote quoting issues — consistent with the `data-path`
pattern used elsewhere in the codebase.

### `infoEl.innerHTML` vs `.textContent`
The `#search-info` element is updated with `.innerHTML` (not `.textContent`) so the filter
chip HTML is rendered. Any user-visible strings inside are `escHtml()`-escaped first.

### Slates stat scope
`stat-slates` counts `slate_count` from **matched** entries only — it reflects the current
search + filter scope, not all available slates.
